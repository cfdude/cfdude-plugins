# pm 0.4.0 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `status: planned` (forward roadmap as ordered backlog), make PROJECT.md re-stamp only on content change, document a roadmap on-ramp, and fix three cleanups — shipping pm 0.4.0.

**Architecture:** All engine logic stays in the single zero-dependency `plugins/pm/scripts/conductor.mjs`. Tests are subprocess-integration (`node:test`) against a temp `CLAUDE_PROJECT_DIR`. Command surfaces are thin markdown files under `plugins/pm/commands/`.

**Tech Stack:** Node 18+ built-ins (`node:fs`, `node:path`, `node:child_process`, `node:url`), `node:test` + `node:assert/strict`. No third-party deps (hard constraint).

**Spec:** `docs/superpowers/specs/2026-06-18-pm-0.4.0-design.md` — read it before starting.

## Global Constraints

- **Zero runtime dependencies.** Node built-ins only. Node 18+.
- **Backward compatibility.** Existing `state.json` keeps working: missing `lane` → `"openspec"`, missing `pmVersion` → `"0.0.0"`, unknown status values tolerated on read.
- **`role` ⊥ `lane`.** Don't conflate; don't touch detour-stack mechanics.
- **Known lanes:** `openspec, superpowers, claude-code, decision, external` (rank: openspec 0, superpowers 1, claude-code 2, decision 3, external 4; unknown 9).
- **Known statuses:** `KNOWN_STATUSES = ["untriaged","queued","active","paused","planned","archived"]`. Validate `--status` on write (`add-epic`); tolerate unknown on read.
- **Flag values must be type-checked as strings.** A valueless CLI flag arrives as boolean `true`; `RegExp.test(true)` returns `true` (it stringifies), so an explicit `typeof === "string"` guard is REQUIRED — the regex alone does not reject non-strings.
- **`planned` epics are actionable-exclusions:** kept out of NEXT UP and out of the brief `lanes:` rollup; surfaced only by a `planned: N — see PROJECT.md` line and the PROJECT.md Epics table.
- **Run tests with:** `node --test plugins/pm/scripts/conductor.test.mjs` (file path — the directory form misbehaves on the installed Node).
- **Commit per task** (conventional commits).
- **Docs + version bump deferred to AFTER Gate 2.** Tasks 1–6 are code/command-surface + tests; the real `plugin.json`/`marketplace.json` stay `0.3.0` during implementation. Task 7 (post-Gate-2) does README/SKILL prose + CHANGELOG + bump.

- [ ] **Step 5: Rewrite `addEpic` with string guards**

Replace the body of `addEpic` (from after `const f = parseFlags(...)` through the success stderr line) with:
```js
  const str = (v) => (typeof v === "string" ? v : undefined); // valueless flags arrive as boolean true
  const id = str(f.id);
  if (!id || !/^[a-z0-9][a-z0-9._-]*$/.test(id)) {
    process.stderr.write("conductor: --id required, format ^[a-z0-9][a-z0-9._-]*$\n"); process.exit(1);
  }
  const lane = str(f.lane);
  if (!lane || !KNOWN_LANES.includes(lane)) {
    process.stderr.write(`conductor: --lane must be one of ${KNOWN_LANES.join("|")}\n`); process.exit(1);
  }
  const status = str(f.status) || "queued";
  if (!KNOWN_STATUSES.includes(status)) {
    process.stderr.write(`conductor: --status must be one of ${KNOWN_STATUSES.join("|")}\n`); process.exit(1);
  }
  const state = loadState();
  if (state.epics.some(e => e.id === id)) {
    process.stderr.write(`conductor: epic '${id}' already exists\n`); process.exit(1);
  }
  const links = (f.link || []).filter(s => typeof s === "string").map(s => {
    const [type, epic, ...rest] = s.split(":");
    const reason = rest.join(":").trim();
    return reason ? { type, epic, reason } : { type, epic };
  });
  const epic = {
    id, title: str(f.title) || id, priority: str(f.priority) || "P?",
    status, role: "epic", lane, links, reconcileNeeded: false,
  };
  if (str(f.plan)) epic.planPath = f.plan;
  state.epics.push(epic);
  saveState(state);
  render();
  process.stderr.write(`conductor: added epic '${id}' (${lane}, ${status})\n`);
```

# pm 0.4.1 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `/pm:upgrade` robust and self-guiding — it refuses (with guidance) instead of silently re-stamping an old version when a newer pm is installed, the SessionStart nudge fires as soon as a newer version is in the cache, and the update sequence is documented.

**Architecture:** A `newestInstalledVersion()` helper scans the plugin cache (env-overridable root) so the engine can see versions newer than the one it's running as. `upgrade()` gains a staleness guard; `brief()`'s nudge keys on newest-installed; docs explain the sequence. Single zero-dep file `plugins/pm/scripts/conductor.mjs`; subprocess `node:test`.

**Tech Stack:** Node 18+ built-ins (`node:fs`, `node:path`, `node:os`, `node:child_process`, `node:url`), `node:test` + `node:assert/strict`. No third-party deps.

**Spec:** `docs/superpowers/specs/2026-06-18-pm-0.4.1-design.md` — read it before starting.

## Global Constraints

- **Zero runtime dependencies.** Node built-ins only. Node 18+.
- **Backward compatible:** existing state.json keeps working; the guard/nudge degrade to no-op / running-version fallback when the cache can't be scanned (`newest === null`).
- **`PM_CACHE_ROOT` is env-overridable** (default `~/.claude/plugins/cache`) for testability; cache layout is `<cacheRoot>/<marketplace>/pm/<version>/.claude-plugin/plugin.json`.
- **Test isolation (C1):** the `run()` helper MUST default `PM_CACHE_ROOT` to an isolated empty temp dir, or `newestInstalledVersion()` scans the developer's real `~/.claude` cache and breaks existing tests.
- **One nudge:** the brief emits at most one upgrade nudge (the new block REPLACES the old running-version block).
- **No schema change, no migration entry.** Patch only.
- **Run tests with:** `node --test plugins/pm/scripts/conductor.test.mjs`.
- **Commit per task.** Docs + version bump deferred to post-Gate-2 (Task 4); `plugin.json`/`marketplace.json` stay 0.4.0 during implementation.

Immediately after `pluginVersion()` (around line 99), add:
```js
/** Highest pm version present in the plugin cache, or null if it can't be determined.
 *  Cache root is env-overridable for testability. Per-entry resilient: one bad
 *  plugin.json doesn't collapse the scan. */
function newestInstalledVersion() {
  const cacheRoot = process.env.PM_CACHE_ROOT || path.join(os.homedir(), ".claude", "plugins", "cache");
  let best = null;
  try {
    for (const mp of fs.readdirSync(cacheRoot, { withFileTypes: true })) {
      if (!mp.isDirectory()) continue;
      const pmDir = path.join(cacheRoot, mp.name, "pm");
      let versions;
      try { versions = fs.readdirSync(pmDir, { withFileTypes: true }); } catch { continue; }
      for (const v of versions) {
        if (!v.isDirectory()) continue;
        const pj = readJSON(path.join(pmDir, v.name, ".claude-plugin", "plugin.json"), null);
        const ver = pj && pj.version ? String(pj.version) : null;
        if (ver && (best === null || cmpVer(ver, best) > 0)) best = ver;
      }
    }
  } catch { /* cache root absent/unreadable → null */ }
  return best;
}
```
